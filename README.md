# 통역 서비스 주간 스케줄러

통역 서비스 관리자가 주간 예약 현황을 관리하고 PNG 이미지로 출력하는 로컬 전용 스케줄러입니다.
서버 없이 `index.html` 파일 하나를 브라우저에서 열면 바로 사용할 수 있습니다.

---

## 실행 방법

`index.html` 파일을 브라우저로 열기 (더블클릭 또는 드래그 앤 드롭)

---

## 주요 기능

### 스케줄 그리드
- **대상 요일**: 월요일 ~ 금요일
- **시간 범위**: 설정 가능 (기본 10:00 ~ 18:00)
- **셀 색상**
  - 초록: 전체 슬롯 예약 가능
  - 주황: 일부 예약됨 (잔여 슬롯 있음)
  - 빨강: 전체 마감
  - 노랑: 점심시간 (예약 불가)

### 예약 블록 관리 (관리자 편집)
- 셀 **클릭** → 예약 블록 +1 (최대 용량까지)
- 셀 **더블클릭** → 해당 슬롯 예약 초기화 (0으로 리셋)

### 점심시간 설정
- 상단 컨트롤바의 🍽 체크박스로 전역 점심시간 활성화/비활성화
- 활성화 시 시작·종료 시간을 드롭다운으로 즉시 변경 가능 (예: 12:00 ~ 13:00)
- ⚙ 날짜별 설정 모달에서 해당 날짜만 점심시간 별도 지정 또는 "점심 없음" 처리 가능
- 점심시간 슬롯은 노란색으로 표시되며 클릭·드래그 불가

### 통역사 슬롯 용량 관리
- 상단 `기본 통역사 수` +/- 버튼으로 전체 기본값 설정
- 각 셀의 ✎ 버튼으로 해당 시간대만 개별 용량 오버라이드 가능

### 슬롯 크기 조절
- 슬롯 **상단 또는 하단** 핸들을 드래그해 슬롯 시간 단위(크기) 조절
  - 하단 핸들: 아래로 드래그 → 현재 슬롯 확장 / 다음 슬롯 축소
  - 상단 핸들: 위로 드래그 → 현재 슬롯 확장 / 이전 슬롯 축소
- 드래그 중 인접 두 슬롯의 크기가 툴팁으로 실시간 표시
- 슬롯에 마우스를 올리면 상·하단 핸들이 나타남 (파란 막대)

### 시간 범위 커스터마이징
- 상단 드롭다운에서 근무 시작·종료 시간 자유 설정
- 날짜별 시간 블록 단위 및 기본 통역사 수 개별 설정 (⚙ 아이콘)

### 휴일 설정
- 각 날짜 헤더의 🏖 아이콘으로 해당 날 휴일 토글

### 탭 전환
| 탭 | 설명 |
|---|---|
| 관리자 편집 뷰 | 블록 설정, 용량 편집 가능한 전체 뷰 |
| 공개 미리보기 | 예약 가능 슬롯만 표시되는 클라이언트용 뷰 |

### 공유 링크
- URL에 전체 스케줄 상태를 인코딩하여 공유 링크 생성
- **LZ-string 압축 + 희소 데이터 최적화** 적용으로 URL 크기 최소화 (v8~)
- 수신자는 읽기 전용 공개 뷰로 현황 확인 가능
- 구버전 링크(v4~v7, Base64 형식)도 자동 호환 지원

### 이미지 출력 (PNG)
- `📷 관리자 뷰 이미지 저장`: 예약 현황·잔여 슬롯 포함
- `📷 공개 뷰 이미지 저장`: 예약 가능 여부만 표시
- 파일명에 해당 주간 날짜 자동 포함
  - 예: `schedule_2026.03.16~03.20_관리자뷰.png`
  - 예: `schedule_2026.03.16~03.20_공개뷰.png`

### 데이터 저장
- 브라우저 LocalStorage에 자동 저장
- 새로고침·재실행 후에도 데이터 유지
- 주차별 독립 저장 (이전 주 데이터 유지)
- 점심시간 설정도 LocalStorage에 저장

## 사용 흐름

1. `index.html` 브라우저로 열기
2. 상단에서 기본 통역사 수, 근무 시간대, 점심시간 설정
3. 이전/다음 주 버튼으로 해당 주로 이동
4. 예약된 슬롯 셀을 클릭해 블록 설정 (더블클릭으로 초기화)
5. 특정 시간대 통역사 수가 다르면 ✎ 버튼으로 개별 설정
6. ⚙ 버튼으로 날짜별 시간 블록·점심시간 개별 설정
7. `공개 미리보기` 탭에서 출력 결과 확인
8. 이미지 저장 버튼으로 PNG 다운로드 또는 공유 링크 생성

## ⚠️ 데이터 저장 및 공유 유의사항

### 1. 관리자 데이터 저장 위치와 초기화 위험

**저장 위치**
- 브라우저의 **LocalStorage**에만 저장됩니다 (서버 없음)
- 저장 키: `sc_weekData`, `sc_defaultCap`, `sc_holidays`, `sc_blockMins`, `sc_startHour`, `sc_endHour`, `sc_lunchEnabled`, `sc_lunchStart`, `sc_lunchEnd`
- 저장 범위: `index.html`을 연 **해당 브라우저 + 해당 경로(Origin)** 에 한정

**데이터가 초기화(삭제)되는 경우**

| 상황 | 결과 |
|---|---|
| 브라우저 설정에서 "사이트 데이터 삭제" 실행 | 모든 데이터 삭제 |
| **시크릿/프라이빗 창**에서 사용 후 탭 닫기 | 세션 종료와 함께 삭제 |
| 브라우저 재설치 또는 프로필 초기화 | 데이터 삭제 |
| `index.html` 파일을 **다른 폴더로 이동** | 경로(Origin)가 바뀌어 기존 데이터 접근 불가 |
| **다른 브라우저**에서 파일 열기 | 해당 브라우저에는 데이터 없음 |
| **다른 기기**에서 파일 열기 | 해당 기기에는 데이터 없음 (동기화 없음) |

> **권장**: 항상 동일한 브라우저, 동일한 경로에서 파일을 열 것. 중요한 데이터는 공유 링크 또는 이미지로 백업해 둘 것.

---

### 2. 공유 링크의 동작 방식

**링크는 매번 새로 생성됩니다 (스냅샷 방식)**

- 공유 링크를 생성하면 **그 시점의 전체 스케줄 상태**를 Base64로 인코딩하여 URL에 포함시킵니다
- URL 구조: `index.html#share=<Base64_인코딩된_전체_데이터>`
- 이후 스케줄을 수정해도 **기존 링크는 수정 전 데이터를 그대로 보여줍니다**
- 스케줄이 변경될 때마다 **새 링크를 다시 생성**하여 공유해야 합니다

**공유 링크로 접속 시**
- 읽기 전용 공개 뷰로만 열립니다 (관리자 편집 불가)
- 공유 링크로 연 상태에서는 **LocalStorage에 저장되지 않습니다**
- 링크 속 데이터는 URL에만 존재하며, 별도로 저장하지 않으면 사라집니다

---

### 3. URL 공유 횟수 및 URL 수 제한

**횟수·갯수 제한 없음**
- 서버가 없으므로 서버 측 생성 제한이 없습니다
- 원하는 만큼 링크를 생성하고 공유할 수 있습니다

**URL 압축 최적화 적용 (v8~)**

v8부터 공유 링크에 **LZ-string 압축 + 희소 데이터 최적화**가 적용됩니다.

| 최적화 기법 | 내용 |
|---|---|
| LZ-string 압축 | JSON을 압축 후 인코딩 (약 60~80% 크기 감소) |
| 희소 데이터 최적화 | 기본값(예약 0건, 기본 통역사 수)은 URL에서 제외 |
| JSON 키 단축 | `capacity→c`, `blocked→b`, `daySettings→d`, `slotDefs→s` 등 축약 |

단, 매우 많은 주차 데이터가 누적된 경우 여전히 URL이 길어질 수 있습니다.

| 환경 | 최대 URL 길이 |
|---|---|
| 대부분의 브라우저 | 2,000 ~ 8,000자 |
| 이메일·메신저 | 자동 줄바꿈 또는 클릭 불가 처리 될 수 있음 |
| 카카오톡·슬랙 등 메신저 | URL 미리보기 생성 실패 가능 |

> **구버전 링크 호환**: v4~v7에서 생성된 Base64 형식 링크도 자동으로 인식하여 정상 동작합니다.

---

### 4. 기타 데이터 저장·관리 유의사항

**LocalStorage 용량 한계**
- 브라우저별로 약 5~10MB 제한이 있습니다
- 주차별 데이터가 누적되면 용량을 초과할 수 있습니다
- 초과 시 저장이 실패하고 데이터가 유실될 수 있습니다

**보안 주의**
- Base64 인코딩은 **암호화가 아닙니다** (누구나 URL을 디코딩해 원본 데이터 열람 가능)
- 공유 링크에는 전체 스케줄 데이터(예약 현황 포함)가 노출됩니다
- 링크를 가진 사람은 누구나 내용을 볼 수 있으므로 **외부 공개에 주의**가 필요합니다

**외부 라이브러리 의존성**
- PNG 이미지 저장 기능은 외부 CDN(`html2canvas`)을 사용합니다
- **인터넷이 없는 환경(오프라인)에서는 이미지 저장이 작동하지 않을 수 있습니다**

**데이터 백업 방법 (현재 기준)**

| 방법 | 특징 |
|---|---|
| 공유 링크 생성 후 저장 | URL 자체가 데이터 백업 역할 |
| PNG 이미지 저장 | 시각적 기록용, 데이터 복원 불가 |
| 브라우저 LocalStorage 직접 복사 | 개발자도구 → Application → LocalStorage에서 수동 복사 가능 |

> **서버나 DB가 없는 구조이므로, 중요한 데이터는 공유 링크를 별도 문서에 보관하거나 PNG로 주기적으로 백업할 것을 권장합니다.**
